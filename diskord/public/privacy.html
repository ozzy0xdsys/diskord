Diskord
Privacy policy

Privacy

Diskord uses client-side encryption (E2EE) for messages and file content inside sessions. The server relays ciphertext and does not store chat history.

What we store

  • Account usernames and password hashes.
  • Active session membership while a session is running.

What we don’t store

  • Message contents or file contents (encrypted payloads are not persisted).
  • Session history after a session ends.

Session deletion

When the session creator ends a session, the session is deleted server-side. Since the server does not persist messages/files, ending a session removes all session data held in memory.
